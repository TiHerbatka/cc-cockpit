## Key decisions made today (with rationale)

- **Browser-first now, Electron as a final milestone.** The user keeps many browser tabs and will accidentally close the cockpit; they want a real standalone app. We keep developing in the browser and convert to Electron later as a clean additive milestone. The architecture is already wrap-ready (factory `createApp`, port-agnostic client, `resolveExecutable`).
- **Rich frontend = Approach B (TPC3), not yet built.** A normalized GUI pane fed by *tailing Claude Code's per-session JSONL* + the existing turn hooks, with the real terminal kept underneath and a per-session GUI <-> terminal toggle. Live activity context (working state, in-flight tool/agent, todo progress) is preserved because it is all in the structured data. Still to design before a spec: pane layout, the "select text -> reference into the next prompt" interaction, and tooltips on detected paths/links.
- **Passive preview is read-only and side-effect-free** (a `peek` message, never `attach`), and must be **sized to the session's PTY grid** then font-scaled — a TUI's byte stream is size-specific, so replaying it into a wrong-sized terminal renders blank/garbled.
- **Temporary sessions:** subfolder-per-temp; folder name is a timestamp, display name becomes Claude's aiTitle; no auto-deletion (they persist and resume).
- **Rename is in-memory** (lost on restart/resume); persisting it is deferred (needs tracking the underlying Claude session id).
- **Trust prompt theory — SUPERSEDED 2026-06-25 (see TODO B2).** We initially thought missing cockpit sessions were caused by Claude's per-folder trust prompt. Follow-up testing on 2026-06-25 **disproved this**: folders under `C:\claude_projects` are already trusted (no prompt appears), yet cockpit-spawned sessions still write **no transcript** to `~/.claude/projects` at all — even while alive and responding. So a fully-working cockpit session (responds, writes files in its cwd, creates its `~/.claude/topics/<id>.json`, has a session id) persists no transcript, which is why nothing shows in Resume, projects read "never used", and temp sessions never auto-name. **This is an open bug — root cause not yet found; next steps + evidence are in TODO B2.** (We did not auto-trust folders / edit `~/.claude.json`.) **— RESOLVED 2026-06-25:** root cause was parent-session env inheritance (`CLAUDE_CODE_CHILD_SESSION=1` leaked into spawned sessions), not the trust prompt. Full writeup: `docs/superpowers/worklog/2026-06-25-b2-no-transcript-rootcause.md`.
- **Assistant may restart the dev server itself** after server-side (`server/*.js`) changes, to test/verify (added to CLAUDE.md conventions). Caveat: restarting the main cockpit kills its live sessions.
